## AI Swarm Orchestration Detail

The AI system follows a **Multi-Agent Swarm** pattern for task execution:

### 1. The Orchestration Cycle
1. **Request Reception**: User query arrives via WebSocket (`aiQuery`).
2. **Analysis (Master Orchestrator)**: The supervisor agent analyzes the intent and context.
3. **Delegation (Handover)**: The orchestrator uses the `delegate_to_agent` tool to transfer control to a specialist.
4. **Execution (Specialist Agent)**: The selected agent (Discovery, Manager, or Analytics) executes its specific tools.
5. **Synthesis**: The results are combined and presented to the user with the agent's identity.

### 2. Specialized Roles
| Agent | Role | Tools |
|---|---|---|
| **Discovery Scout** | Search & Discovery | `searchEvents`, `getEventDetails` |
| **Event Manager** | CRUD Operations | `createEvent`, `updateEvent`, `deleteEvent`, `registerSomeone` |
| **Data Insight** | Analytics | `getEventsStats` |

### 3. Personal Concierge Mode
When a user views a specific event, the AI service automatically injects the event's data into the context, transforming the active agent into a **Personal Concierge** for that specific мероприятие.

## Level 4: Code (Key Patterns)

### WebSocket Handover Pattern
```typescript
// AI Service handles parallel tool execution and agent delegation
async query(userId: string, dto: AiQueryDto) {
  const activeAgent = await this.orchestrate(userQuery);
  const result = await this.execute(activeAgent, userQuery);
  return { ...result, agent: activeAgent.name };
}
```

```
[Client WS] → [Handshake Auth] → [JWT Validation] → [Gateway] → [Swarm Service] → [PostgreSQL]
```
